  1. Design the component using CSS and HTML.
  2. Make it work with the Array<Day> generated in the AppComponent (days)
  3. Add pagination to the component and the buttons to control it. Look at the shadows that appears in both sides if there are more pagination.
  4. Add an input attribute [multiple]="boolean" to the component. If multiple true, the component will allow selecting more than one date, in other case, only once.
  5. Make it compatible with NgModel. <app-selector [(ngModel)]="model"></app-selector>
  6. Make it responsive. If the browser is resized or the parent container decreases, the component should adapt to its parent. You should avoid reseting pagination.
    Example:
    - Inital window: [1,2,3,4,5]
    - Click next: [6,7,8,9,10]
    - Resize decreasing width: [7,8,9]
    Notice that pagination fits with the number of elements in the window, in this case, 3 elements can be shown in the window, for this reason it shows [7,8,9], if 4 elements fit, it should show [5,6,7,8].
  7. Do test cases you consider necessary in order to check the component works properly.
  8. Run npm run lint and fix lint issues.
  9. Generate a git patch with all changes and send it to sergio@tulotero.es.

If you have problems to figure the exercise out, you can send the patch even if you aren't able to finish any of the latest points.

Good luck! :-)
